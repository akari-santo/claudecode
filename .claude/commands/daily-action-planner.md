# Daily Action Planner — 案件NA × カレンダーの1日アクションプラン生成

## 概要

Notion案件管理DBに記載されたNA（Next Action）群と、Googleカレンダーの予定を突合し、
「今日やるべきこと」を優先度付きで抽出し、MTGの隙間時間に割り当てた実行可能なプランを生成する。

「全部のNAを一覧にする」のではなく、**今日動かすべきものだけ**を絞り込むことが最大の価値。

---

## インプット

1. **Notion案件管理DB（テキスト）**: タブ区切りまたはコピペされた案件情報。各案件に以下を含む：
   - 案件名、期間、ステータス（`8_契約中`, `6_リーガルチェック`, `5_稟議`, `4_金額交渉中`, `3_review`, `2_qualify`, `1_アポ獲得済み`）
   - 金額
   - NA（Next Action）リスト
   - 現状のステータス、次回MTG情報

2. **Googleカレンダー（スクリーンショット）**: 当日の予定が見える画像。MTG名・時間帯を読み取る。

---

## 処理フロー

### Step 1: カレンダーの読み取り

画像からMTGの時間帯を読み取り、以下を構造化する：
- 各MTGの開始・終了時刻と名称
- **空き時間ブロック**を特定する（MTGとMTGの間、朝イチ、MTG後の時間帯）
- 移動時間やランチ（12:00-13:00目安）も考慮する

### Step 2: NAの抽出とフィルタリング

案件データから全NAを抽出し、以下の基準で**今日やるべきNA**をフィルタリングする：

**抽出対象（今日のアクション候補）**:
- NA欄に「今日の日付（例: 3/19）」が明記されているもの → 最優先で抽出
- NA欄に「今週中」「今週」「急ぎ」等の記載があるもの
- 期日が今日〜3日以内のもの
- 「電話」「メール」「Slack連絡」「リマインド」「日程調整」等、5-15分で完了する軽タスク
- MTGの事前準備が必要なもの（カレンダーに該当MTGがある場合）

**除外対象**:
- 「先方ボール」と明記されているもの（ただし期限超過なら催促として復活）
- 期日が1週間以上先で、今日着手する必然性がないもの
- エンジニアの作業待ちなど、自分では動かせないもの

### Step 2.5: 所要時間の見積もり

NAごとに以下の基準値テーブルを参照して所要時間を見積もる。
基準値はコンテキストスイッチ（準備・後処理）込みの実時間。

| タスクカテゴリ | 基準時間 | 例 |
|--------------|---------|-----|
| 日程調整（メール/Slack送信） | 10分 | 「次回日程調整」「リマインドメール」 |
| 報告書・議事録の送付（作成済みのものを送る） | 10分 | 「報告書送付」「議事録送付」 |
| 議事録作成（メモから構造化） | 15分 | 「議事録作成」「外報作成」 |
| SoW作成 by Claude（Claudeに依頼して生成→軽微修正） | 20分 | 「SoW作成 by Claude」 |
| スライドへの転記・SoW転記（生成物をpptx/Googleスライドに反映） | 45分 | 「スライド化」「pptxに転記」 |
| 見積書作成（費用計算＋書式整え） | 30分 | 「正式見積書作成」「概算資料作成」 |
| 費用見積もり・内訳検討（積算ロジックの検討含む） | 60分 | 「費用内訳整理」「工数見積もり」 |
| 契約書修正・法務依頼 | 15分 | 「法務にSlack」「契約書修正依頼」 |
| 電話（状況確認・軽い相談） | 10分 | 「川中さんに電話」「リマインド電話」 |
| 質問回答（Excel/メールで回答作成） | 30分 | 「別紙質問回答」「和田様の質問回答」 |
| 社内Slackで引き継ぎ・依頼 | 10分 | 「竹辻さんに引き継ぎ」 |
| 提案資料・スライド作成（ゼロから） | 90分 | 「提案資料作成」「概算資料新規作成」 |
| レビュー（他者の成果物を確認） | 20分 | 「review」「概算レビュー」 |

上記に当てはまらないタスクは、文脈から最も近いカテゴリの基準値を適用する。

### Step 3: 優先度スコアリング

抽出したNAに対して、以下の重み付けで優先度を判定する：

| 判断軸 | 重み | 高スコア例 |
|--------|------|-----------|
| ステータスの進行度 | ★★★ | リーガルチェック・稟議 > 金額交渉 > review > qualify |
| 期日の切迫度 | ★★★ | 今日マスト > 今週中 > 来週 |
| 所要時間 × 空き時間のフィット | ★★ | 5分タスクは隙間に、60分タスクはまとまった空きに |
| 案件金額 | ★ | 大型案件のNA > 小型案件のNA |
| ブロッカー解除効果 | ★★ | これをやらないと他の人が止まるもの |

**優先度ラベル**:
- 🔴 **MUST**（今日やらないとマイルストーンに影響）
- 🟡 **SHOULD**（今日やると案件が前に進む）
- 🟢 **NICE**（今日やれたらベター、来週でも致命的ではない）

### Step 4: 時間割への割り当て

空き時間ブロックにタスクを割り当てる。以下の原則に従う：

1. **MTG前の10-15分**: そのMTGに関連する事前準備タスクを優先配置
2. **短い隙間（5-20分）**: 電話・メール・Slack連絡・リマインド等の軽タスク
3. **30分以上のまとまった空き**: 資料作成・SoW作成・レビュー等の重タスク
4. **MTG直後の5-10分**: 直前のMTGに関連するフォローアップ（議事録送付、NA共有等）
5. **🔴 MUSTタスクは必ずどこかの時間帯に割り当てる**（溢れる場合はMTG後の時間に）

## 出力フォーマット

以下のMarkdown構造で出力する。会話内テキストとして直接出力し、ファイルは生成しない。

### アクション方向フラグ

各タスクの先頭に、内部向け/外部向けを示すフラグを付与する：
- `→外` : 顧客・外部パートナーへのアクション（メール送付、電話、資料送付、質問回答等）
- `→内` : 社内アクション（法務依頼、エンジニア連携、社内レビュー、引き継ぎ、Claude作業等）

外部向けアクションは先方の営業時間内（9:00-18:00）に実行するのが望ましいため、
空き時間の割り当て時にも優先的に日中の枠に配置する。

### ステップ分解

各タスクには（）書きで具体的な実行ステップを付記する。
ユーザーがタスクを見た瞬間に「最初に何を開いて何をすればいいか」がわかる粒度にする。

**ステップ分解の例**:
- `→外 日本セイフティー 別紙質問回答送付（Excelファイルを開く→回答欄を埋める→メールに添付して浅倉課長宛に送信）`
- `→内 大東建託Phase2 SoW作成（Claudeにsow-generator-pocで生成依頼→出力を確認・微修正→Googleスライドに転記）`
- `→外 積水ハウス 正式見積書作成（前回概算をベースに期間・金額を確定→見積書テンプレに転記→和田様宛メールで送付）`

ステップは2-4個程度。1ステップが1アクション（1つのツールを開く、1通送る等）に対応するのが理想。

## 判断の補足ルール

### ステータス別の「今日動かすべき度」

案件のステータスは進行度が高いほど、NAを今日中に処理する重要性が高い。
遅延は契約・売上に直結するため、以下の順で重み付けする：

1. **6_リーガルチェック / 5_稟議**: 契約直前。書類送付・回答・確認は即日対応が基本
2. **4_金額交渉中**: 先方の温度感が冷めるリスク。連絡・回答は翌日持ち越し非推奨
3. **8_契約中**（納品・検収フェーズ）: 報告書送付・テスト・請求書等の期日遵守
4. **8_契約中**（開発進行中）: エンジニア連携・進捗確認は定期的に
5. **3_review**: 提案準備・SoW作成。まとまった時間が必要だが急務度は中
6. **2_qualify / 1_アポ**: 日程調整・連絡は軽タスクだが放置するとフェードアウト

### 「チェックマーク付きNA」の扱い

NA欄に ` ` や `✅` が付いているものは**未完了だが着手済み or 期日付き**のアクション。
日付が今日以前のものは「期日超過」として🔴 MUSTに格上げする。

### カレンダー上のMTGと案件の紐付け

MTG名に含まれるクライアント名・テーマ名を案件データと照合し、
そのMTGの事前準備として必要なNAがあれば、MTG前の空き時間に自動配置する。

### 案件管理MTG（竹辻・吉岡）の活用

カレンダーに「案件管理」系のMTGがある場合、そのMTGで共有・相談すべき事項を
タイムラインの該当箇所に付記する（例: 「竹辻さんに引き継ぐ案件の確認」等）。

## 出力時の注意

- **情報の落とし方を明示する**: 全案件のうち何件が今日のアクション対象で、何件はスキップしたかをサマリーに含める
- **所要時間はStep 2.5の基準値テーブルに従う**: テーブルにないタスクは最も近いカテゴリを適用。勝手に短く見積もらない
- **空き時間を100%埋めない**: 30分以上の空きブロックでも70-80%の充填率にとどめ、バッファを残す
- **MTG後の処理時間を確保する**: 連続MTGの間に5-10分のフォローアップ時間を想定する
- **→外タスクは日中（9:00-18:00）に優先配置する**: 先方の営業時間内に送付・連絡を完了させる
- **ステップ分解は省略しない**: タイムライン内の全タスクに（）書きでステップを付記する。「何を開いて何をする」が一目でわかる粒度を保つ
